# 添加推理模型总排行榜和开源排行榜计划

## 目标

在现有通用排行榜的基础上，新增抓取推理模型总排行榜和开源排行榜数据，并提供前端切换功能。

## 网站结构分析

通过浏览器截图和网页内容分析，确认以下信息：

* **通用排行榜**：`https://www.superclueai.com/generalpage`（已实现）
* **推理模型总排行榜**：`https://www.superclueai.com/reasoning-models`
* **开源排行榜**：`https://www.superclueai.com/open-models`

### 通用排行榜表格结构（12列）
排名、模型名称、机构、开/闭源、总分、数学推理、幻觉控制、科学推理、指令遵循、代码生成、智能体任务规划

### 推理模型总排行榜表格结构（推测：12列）
排名、模型名称、机构、开/闭源、总分、数学推理、幻觉控制、科学推理、指令遵循、代码生成、智能体任务规划
（与通用榜结构相同，仅展示推理类模型）

### 开源排行榜表格结构（推测：12列）
排名、模型名称、机构、开/闭源、总分、数学推理、幻觉控制、科学推理、指令遵循、代码生成、智能体任务规划
（与通用榜结构相同，仅展示开源模型）

> 注：推理模型和开源排行榜的实际表格结构需要验证，可能与通用榜略有差异

## 步骤

### 1. 更新爬虫脚本 (`scraper/scrape.py`)

* 扩展 `SOURCES` 配置，新增推理和开源两个榜单

* 为每个榜单定义 `board_type` 标识（general, reasoning, open）

* 修改 `scrape_source` 函数，增加 `board_type` 参数

* 更新 `parse_superclue_table` 函数，提取表格中所有维度数据

* 修改 `save_data` 函数，支持多榜单数据保存

* 修改 `main` 函数，遍历所有榜单配置并依次抓取

### 2. 更新数据格式

将 `models.json` 结构改为支持多榜单：

```json
{
  "update_time": "2026年3月",
  "leaderboards": {
    "general": {
      "name": "通用排行榜",
      "models": [...]
    },
    "reasoning": {
      "name": "推理模型总排行榜",
      "models": [...]
    },
    "open": {
      "name": "开源排行榜",
      "models": [...]
    }
  }
}
```

### 3. 更新前端 HTML (`index.html`)

* 在导航栏"排行榜"链接下添加下拉菜单，包含三个子链接：通用排行榜、推理模型总排行榜、开源排行榜

* 每个子链接通过锚点 (#general, #reasoning, #open) 实现快速跳转

* 在搜索栏和表格之间添加排行榜切换按钮组（作为导航栏的补充）

* 包含三个按钮：通用排行榜、推理模型总排行榜、开源排行榜

* 添加 `data-board` 属性用于JS识别

* 保持现有表格结构不变（12列）

### 4. 更新前端 JS (`js/main.js`)

* 添加 `currentBoard` 状态变量，默认为 'general'

* 修改 `fetchData` 函数，加载完整的 `leaderboards` 对象

* 添加 `switchBoard` 函数处理榜单切换

* 修改渲染逻辑，根据 `currentBoard` 显示对应榜单的数据

* 保持排序和搜索功能在各榜单中正常工作

### 5. 更新 CSS 样式 (`css/style.css`)

* 添加排行榜切换按钮组样式（类似标签页切换效果）

* 按钮选中状态使用蓝色高亮

* 确保移动端响应式布局正常

### 6. 测试验证

* 本地运行爬虫脚本验证三个榜单数据抓取

* 启动本地服务验证前端榜单切换功能

* 测试搜索和排序功能在各榜单中正常工作
