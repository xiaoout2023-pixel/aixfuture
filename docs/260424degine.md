AIXfuture Portal MVP Design (v0.1)

日期：2026-04-24\
范围：多维度大模型排名门户网站（初版MVP）

## 1. 背景与目标

**MVP愿景**：快速上线一个展示大模型多维度排名的门户网站，让用户可以在一张表格中比较不同模型的性能、价格、语言能力等核心指标。

**用户目标**：做一个 **AIXfuture 门户网站**，风格 **简约**。核心内容是多类 AI 资产的 **榜单聚合**（不自造评分体系）

**核心功能**：

- **多维度排名表**：展示模型名称、厂商、类型、语言能力、评分、价格、参数量、特殊能力
- **搜索与排序**：用户可以按列排序或搜索模型名称/厂商
- **数据抓取**：从公开榜单或官方页面抓取模型信息

***

### 5.2 搜索和排序

**功能**：

- 点击列名可排序
- 搜索框支持模糊匹配模型名称和厂商
- 排序算法：数字列按大小排序，字符串列按字母排序

***

### 5.3 数据抓取

**目标**：抓取公开榜单和官方页面，生成 JSON 文件

**数据源（示例）**：

- [数字排行榜（https://superclueai.com/generalpage）](https://superclueai.com/generalpage)
- 视频生成榜([https://superclueai.com/benchmarkselection?category=multimodal)](https://superclueai.com/benchmarkselection?category=multimodal)

**抓取方式**：

- Python + requests + BeautifulSoup4 / playwright
- 定期（每15天）抓取或手动运行生成 `models.json`
- 生成字段对应 JSON 数据模型

**抓取脚本逻辑**：

1. 遍历每个数据源
2. 抓取排名
   模型
   机构
   总分
   完成数
   完成率
   异常数
   异常率
   空补丁数
   空补丁率
   解决数
   转化率
   解决率
   平均测试时长(s)
   属地
   使用方式等
3. 如抓取不到时，显示上一次已抓取的数据
4. 处理异常（页面未加载、字段缺失）
5. 写入 `models.json`

***

## 6. 技术方案

### 6.1 前端

- HTML + CSS + JS
- 表格展示 JSON 数据
- 支持排序和搜索
- 移动端响应式

### 6.2 后端

- 可选：静态 JSON 提供给前端
- 或使用 Python Flask / FastAPI 提供 API

### 6.3 数据抓取

- Python 脚本生成 JSON
- 定时任务可选

***

## 7. MVP上线计划（2小时目标）

1. 编写抓取脚本生成示例 JSON
2. 编写前端表格页面读取 JSON 并渲染
3. 实现搜索和列排序
4. 部署到静态站点（Vercel / Netlify）
